# GitLab 工作汇总

## 必要输入

- 确认 GitLab 地址、统计开始日期、结束日期和时区。
- 默认周期为当前时区自然周周一至当天。
- 确认报告中使用的项目名称；代码仓库名与业务项目名不一致时，以用户确认结果为准。
- 通过当前进程的 `GITLAB_TOKEN` 环境变量提供访问令牌。

## 工作流

1. 运行 `scripts/fetch_gitlab_work.py` 获取推送事件、提交、文件列表和 diff 摘要。
2. 按项目、提交 SHA 和事件时间去重，避免重复推送导致重复统计。
3. 对“同上”“同步代码”“问题修复”等模糊提交使用单提交模式读取更多 diff。
4. 按业务能力归并前端、后端、数据库、GIS 和样式改动。
5. 读取 [references/work-record-style.md](references/work-record-style.md) 确定句式和输出格式。
6. 对照原始提交检查事实、覆盖范围、时间边界和重复内容。

## 采集示例

```powershell
$env:GITLAB_TOKEN = '<personal-access-token>'
python scripts/fetch_gitlab_work.py `
  --base-url 'https://gitlab.example.com' `
  --start-date 2026-07-06 `
  --end-date 2026-07-10
Remove-Item Env:GITLAB_TOKEN
```

仅当 GitLab 使用自签名证书且用户接受风险时传入 `--insecure`。

## 归并规则

- 同一业务能力的页面、接口、数据库和样式改动合并为一条成果。
- 公共组件仓库和业务仓库中的同一份同步代码只记录一次。
- 不同业务模块分别成条，不按提交时间强行合并。
- 小型字段、图标和布局调整并入所属功能，不拆成低价值条目。
- 只有代码和 diff 明确体现时才使用“新增”“完成”“支持”等确定表达。
- 没有证据时不声称已经部署、上线、联调或测试通过。

## 安全与输出

- 不把令牌写入 Skill、脚本、命令输出、缓存文件或报告。
- 默认不展示提交 SHA、仓库路径、文件名和技术实现细节。
- 不仅凭提交标题推断工作内容。
- 报告每一项都应能对应提交、文件或代码逻辑。
- 删除没有事实依据的“提升、赋能、保障、闭环”等效果描述。
- 说明推送事件口径不包含未推送的本地提交，也可能不包含他人代为推送的提交。
