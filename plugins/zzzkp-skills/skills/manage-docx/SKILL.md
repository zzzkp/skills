# DOCX 文档管理

## 目标

安全地处理 DOCX 内容和版式，并在交付前检查文本、表格、图示、分页和整体可读性。

## 工作流

1. 确认输入文件、用户目标和输出位置。
2. 优先渲染文档进行可视化审阅；需要命令和依赖说明时读取 [references/rendering.md](references/rendering.md)。
3. 使用项目已有工具或 `python-docx` 读取、编辑或创建标题、样式、表格和列表。
4. 只修改用户要求的内容，尽量保留现有样式、页眉页脚、节和分页设置。
5. 每次实质性修改后重新渲染并检查受影响页面。
6. 无法渲染时提取文本进行结构检查，并明确说明未验证的布局风险。

## 文件与依赖规则

- 优先使用用户指定的输出路径；未指定时，在输入文件附近生成具有描述性的副本，不覆盖原文件。
- 临时文件放在当前任务的独立临时目录，只清理本次创建的文件。
- 不删除、移动或覆盖用户原始文件，除非用户明确要求。
- 缺少渲染工具或 Python 包时，先报告缺失项和影响；获得授权后才能安装。
- 使用 `scripts/render_docx.py` 时，先确认其依赖在当前环境可用。

## 质量要求

- 保持标题层级、字体、间距、页边距和列表样式一致。
- 检查文本裁切、元素重叠、表格断裂、字符不可读和异常空白页。
- 确认图表、图片和表格清晰可读且对齐正确。
- 不遗留工具令牌、占位字符串、调试内容或重复渲染文件。
- 最终说明修改内容、输出文件和未能验证的布局风险。
