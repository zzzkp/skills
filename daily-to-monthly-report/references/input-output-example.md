# 输入输出样例（来自“新建 文本文档.txt”）

## 目标

- `skills目标`：汇总日报为月报
- `skills名称`：自由发挥（已在实现中规范为 `daily-to-monthly-report`）

## 输入示例片段

```text
2026-02-02  浙江电信：根据安全扫描文档完成应用服务器漏洞整改，涉及 Redis、Zookeeper 等组件的安全配置优化与调整
2026-02-03  河南电信：页面调整。更新节假日保障页面中蛇年元素图标为马年元素，并去除所有页面中的悬浮语音助手图标
2026-02-03  浙江电信：导出 2025 年平台功能使用日志，统计各功能使用次数及使用人数
2026-02-10  江苏电信：FAST-站址运营评估模块详情页及导出功能优化，新增字段并完善导出内容
2026-02-10  浙江电信：协助排查新申请大模型服务无法正常调用问题，定位并排除 Nginx 转发配置异常
2026-02-23  （空内容，且标记为“否”）
2026-02-24  浙江电信：完成 GeoServer 高铁线路 4G/5G 图层编制、发布与测试
2026-02-26  浙江电信：高铁分析模块页面优化，GIS 组件新增高铁线路图层加载与渲染，并支持点击线路弹出路段详情
```

## 输出示例结构

```markdown
一、浙江电信
1. ...
2. ...

二、河南电信
1. ...

三、江苏电信
1. ...
2. ...

天数汇总（按总天数22天，每日涉及多个项目则均分）：
河南电信：3天
江苏电信：7天
```

## 天数计算口径

1. 仅统计有效记录（排除“否”与空内容）。
2. 某日期若涉及 `N` 个项目，则每个项目记 `1/N` 天。
3. 项目月度天数为该项目在全部有效日期的累计值。
4. 若用户明确给出总天数口径（如 22 天），按该口径解释并均分；未给出时按实际有效日期计算。

## 文风约束

- 用“完成/优化/修复/排查/开发/部署”等结果导向动词。
- 合并同类项，避免逐日重复流水账。
- 保留关键对象（模块名、系统名、接口名、GIS/GeoServer 等）。
