# Daily To Monthly Report

## 处理目标

- 读取日报明细（至少包含日期、工作内容、有效标记）。
- 输出月报正文（按项目分组，条目化总结）。
- 输出天数汇总（同日多项目按均分计入）。

## 执行步骤

1. 预处理输入：解析表格行，保留字段 `日期`、`工作内容`、`是否有效`。
2. 过滤无效记录：跳过 `是否有效=否`、工作内容为空、仅占位内容的行。
3. 拆分多任务内容：当同一行包含多段“XX电信：...”时，拆分为多条任务。
4. 项目归类：按“浙江电信/河南电信/江苏电信”等项目名称聚合。
5. 月度归并：将同类任务跨日期合并为一条月报成果，保持“动作+结果”表达，避免逐日流水账重复。
6. 生成正文结构：按项目输出一级标题，标题下输出编号条目（`1. 2. 3.`）。
7. 计算天数汇总：
   - 每个有效日期先识别当天涉及的项目集合，数量记为 `N`。
   - 当天每个项目记 `1/N` 天。
   - 月度项目天数为该项目在全部有效日期上的累计值。
   - 若用户提供“总天数”口径（如 22 天），以该口径为准并按同样规则均分。
8. 自检输出：确认项目标题、编号、语义去重、天数逻辑一致。

## 输出格式

按以下格式组织输出：

```markdown
一、浙江电信
1. ...
2. ...

二、河南电信
1. ...

三、江苏电信
1. ...
2. ...

天数汇总（按总天数X天，每日涉及多个项目则均分）：
河南电信：Y天
江苏电信：Z天
```

## 归并规则

- 将“页面调整/问题修复/优化”归并为同类项时，保留关键系统名与结果。
- 将“开发 + 联调 + 部署”可合并为一条端到端交付项。
- 将“GIS图层加载、渲染、弹窗联动”可合并为地图能力建设项。
- 相同任务在不同日期重复出现时，合并表述并体现持续优化，不重复计条。
