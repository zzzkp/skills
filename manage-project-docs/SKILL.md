# 项目文档管理

## 目标

按“项目总览 → 模块总览 → 专题文档”组织项目知识，渐进读取必要内容，避免重复文档和巨大 README。

## 定位项目

1. 优先使用用户明确指定的文档目录和项目名。
2. 用户指定环境变量时读取变量值并确认目录存在；没有指定时可以检查 `PROJECT_DOCS_ROOT`。
3. 当前任务已明确项目路径时直接使用该范围，不强制要求环境变量。
4. 项目名存在多个候选或无法可靠匹配时，列出少量候选并向用户确认。
5. 查阅或修改任务找不到项目目录时，不自行创建；生成新项目文档时按用户要求创建。

## 目录模型

```text
<docs-root>/<project>/
├── README.md
├── frontend/README.md
├── backend/README.md
├── database/README.md
├── api/README.md
├── deployment/README.md
└── operations/README.md
```

- 项目根 `README.md` 只保留项目概览、代码路径、一级索引和维护约定。
- 模块 `README.md` 只保留模块概览、关键约定和专题索引。
- 详细内容拆分为模块目录下单一主题的专题文档。
- 新增前检查已有索引和近似主题，避免 `backend/`、`后端/`、`server/` 等重复结构。

## 标准工作流

1. 确认文档根目录、项目目录和任务类型。
2. 先读取项目根 `README.md` 或项目文件列表。
3. 根据问题读取相关模块 `README.md`，需要细节时再读取少量专题文档。
4. 优先更新已有文档；只有现有主题无法承载时才新增模块或专题。
5. 新增、删除或重命名文档后，同步更新直接上级索引和所有相关链接。
6. 完成后检查主题边界、相对链接、代码路径和修改范围。

## 路径规则

- 在项目根 `README.md` 中记录项目代码路径。
- 文档引用代码文件时，以项目代码路径为基准写代码仓库内相对路径。
- 文档之间的 Markdown 链接以当前文档位置为基准。
- 不在专题文档中重复写绝对代码路径。
- 删除或重命名文档时修正全部入站链接。

## 创建与修改规则

- 保留项目已有术语、标题层级、命名和链接风格。
- 只修改用户请求相关的项目、模块和主题。
- 新增一级模块时创建模块目录和模块 `README.md`，并更新项目根索引。
- 新增专题时放入对应模块目录，并更新模块索引。
- 一个专题只覆盖一个主要主题；内容跨多个读者或场景时继续拆分。
- 需要新文档骨架时使用 `assets/templates/` 中的模板。

## 完成检查

- 检查新增、删除和重命名是否同步更新索引。
- 检查 Markdown 相对链接能否从当前文件位置解析。
- 检查是否存在重复主题、重复模块或巨大 README。
- 检查代码路径是否以项目代码路径为基准。
- 检查是否修改了用户未要求的项目或模块。
- 回答查阅问题时说明信息来自哪个项目和哪些文档。
